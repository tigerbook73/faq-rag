# 需求文档：back-to-single-user

## 目标

将应用简化为单用户 FAQ RAG 工具，去除所有身份验证、多用户管理及管理员功能。应用无需登录，直接打开即可使用。

---

## 用户可见变化

### 删除

- 登录页 — 应用不再需要登录
- 退出登录按钮
- 管理员后台（`/admin`）
- 管理员用户管理
- 管理员文档管理
- 文档公开 / 私有可见性切换
- 聊天界面中的"公开文档"选择面板

### 保留（用户体验不变）

- 聊天界面 — 提问、流式回答、引用来源
- 会话列表 — 创建、重命名、删除、导出会话
- 知识库页面 — 上传、删除、重新索引文档
- LLM 提供商选择（Claude / DeepSeek / OpenAI）
- 文档状态指示（pending / indexing / indexed / failed）
- 多语言查询支持（中文 / 英文）

### 入口行为

- 应用直接打开聊天界面，不跳转登录页

---

## 外部依赖

| 依赖                                       | 用途                       | 变更     |
| ------------------------------------------ | -------------------------- | -------- |
| Supabase Database（PostgreSQL + pgvector） | 存储文档、向量块、会话数据 | 不变     |
| Supabase Storage                           | 存储上传的文档文件         | 不变     |
| Supabase Auth                              | 用户身份验证               | **移除** |
| Anthropic API                              | Claude LLM                 | 不变     |
| DeepSeek API                               | DeepSeek LLM + 查询扩展    | 不变     |
| OpenAI API                                 | OpenAI LLM + 可选向量化    | 不变     |
