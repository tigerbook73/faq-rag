## 十、生产加固

### 10-B ✅ 结构化日志 + 请求链路追踪 【已完成】

|              |                                               |
| ------------ | --------------------------------------------- |
| **难度**     | ⭐⭐ 中等                                     |
| **预计工时** | 3–5 小时                                      |
| **核心技能** | 结构化日志（pino）、请求 traceId、可观测性    |

**现状**：散落的 `console.log`，无法按请求过滤日志，无法追踪一次检索的各阶段耗时。

**目标**：

1. 引入 `pino` + `pino-pretty`，统一日志格式（JSON 结构，含 `level`、`time`、`traceId`）。
2. 每次 `POST /api/chat` 生成 `traceId = crypto.randomUUID()`，透传进 `retrieve()`、`rerankChunks()` 各层。
3. 记录关键耗时：`retrieval_ms`、`rerank_ms`、`llm_first_token_ms`、`retrieval_top_score`。

**学习点**：结构化日志 vs. 非结构化日志，可观测性三支柱（Logging / Metrics / Tracing）基础。

### 10-C 身份验证（NextAuth.js）

|              |                                       |
| ------------ | ------------------------------------- |
| **难度**     | ⭐⭐⭐ 中等偏高                       |
| **预计工时** | 4–8 小时                              |
| **核心技能** | NextAuth.js v5、Session 管理、OAuth   |

**目标**：为本地部署添加基本访问控制，防止知识库和对话历史暴露给未授权用户。

**方案**：Credentials Provider（用户名/密码，简单哈希存 `.env`）+ 可选 GitHub OAuth。

- `middleware.ts` 保护 `/chat/*` 和 `/knowledge` 路由
- 登录页 `/auth/signin`
- Server Components 通过 `auth()` helper 读取 Session
