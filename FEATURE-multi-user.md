## 增加多用户支持

### 目标

让 FAQ RAG 支持多个用户共同使用，同时保证每个用户的聊天记录和私有文档相互隔离。用户可以选择把自己的文档公开给其他用户使用，其他用户只有在主动选择后，才会在问答检索中使用这些公开文档。

### 角色

系统包含两类用户：

- 普通用户：管理自己的聊天记录和文档，可以使用自己的文档和已选择的公开文档进行问答。
- 管理员：继承普通用户能力，并可以管理其他用户和全站文档。

默认需要创建以下账号，方便演示和测试：

- `admin@test.com` / `admin@123`，管理员
- `user1@test.com` / `user1@123`，普通用户
- `user2@test.com` / `user2@123`，普通用户

登录页需要展示这些默认账号信息，方便直接试用。

### 普通用户 Use Cases

#### 登录

- 普通用户可以使用邮箱和密码登录系统。
- 登录成功后，用户只能进入自己有权限访问的页面和数据范围。
- 登录页展示默认测试账号和密码，方便演示和测试。
- 普通用户暂不支持修改自己的邮箱或密码。

#### 管理自己的聊天记录

- 普通用户可以查看自己的聊天列表。
- 普通用户可以创建新聊天。
- 普通用户可以打开自己的历史聊天。
- 普通用户可以更新自己的聊天记录，包括标题和消息内容。
- 普通用户可以删除自己的聊天。
- 普通用户不能看到、打开、更新或删除其他用户的聊天记录。
- 即使普通用户知道其他用户的聊天 ID，也不能通过 URL 或 API 访问。

#### 上传和管理自己的文档

- 普通用户可以上传文档。
- 上传文档必须归属于当前登录用户。
- 不同用户允许上传内容相同的文件。
- 上传文档时，普通用户可以选择文档可见性：
  - `private`：仅上传者本人可见、可用于问答。
  - `public`：上传者本人可见、可用于问答；其他用户可以看到并选择是否用于自己的问答。
- 普通用户可以查看自己的文档列表。
- 普通用户可以修改自己文档的可见性。
- 普通用户可以删除自己的文档。
- 普通用户不能查看、修改或删除其他用户的私有文档。
- 普通用户不能修改或删除其他用户的公开文档。

#### 选择其他用户的公开文档

- 普通用户可以查看其他用户发布的公开文档列表。
- 其他用户的公开文档不会默认参与当前用户的问答。
- 普通用户可以主动勾选其他用户的公开文档。
- 普通用户可以取消勾选已选择的公开文档。
- 普通用户自己的公开文档不需要勾选，始终可用于自己的问答。
- 普通用户不能勾选私有文档。
- 普通用户不能勾选已删除、未索引完成或索引失败的文档作为有效问答来源。

#### 提问和获取回答

- 普通用户可以基于自己有权使用的文档提问。
- 回答中的引用来源只能来自当前用户有权使用的文档。
- 普通用户不能通过提问检索到其他用户的私有文档。
- 普通用户不能通过提问检索到未勾选的其他用户公开文档。
- 用户删除文档、取消勾选公开文档或文档可见性发生变化后，后续问答必须立即遵守新的权限范围。

### 管理员 Use Cases

管理员继承普通用户的全部能力。除普通用户能力外，管理员还具备以下差异化能力。

#### 管理用户

- 管理员可以查看用户列表。
- 管理员可以创建普通用户。
- 管理员可以删除普通用户。
- 管理员不能删除自己的账号。
- 删除用户前需要有明确确认。
- 删除用户后，该用户相关内容必须被清理，详见“系统自动清理规则”。
- 系统暂不记录管理员删除用户的审计日志。

#### 管理全站文档

- 管理员可以查看所有用户上传的文档。
- 管理员可以删除任意用户上传的文档。
- 管理员删除文档前需要有明确确认。
- 管理员删除文档后，该文档不能再被任何用户查看、选择或用于问答。
- 系统暂不记录管理员删除文档的审计日志。

#### 管理员默认边界

- 管理员默认也只使用自己的聊天记录。
- 管理员不允许查看其他用户的聊天内容。
- 管理员在普通问答流程中，使用规则与普通用户一致：自己的文档和自己已选择的公开文档参与检索。

### 核心业务规则

#### 文档可见性

- `private` 文档只属于上传者本人。
- `public` 文档属于上传者本人，同时可以被其他用户发现和选择。
- 文档可见性只影响其他用户能否发现和选择该文档，不影响上传者本人使用该文档。

#### 问答检索范围

用户提问时，可使用的文档范围为：

- 自己上传的 `private` 文档。
- 自己上传的 `public` 文档。
- 自己已勾选的其他用户 `public` 文档。

不允许进入问答检索范围的文档：

- 其他用户的 `private` 文档。
- 其他用户未被当前用户勾选的 `public` 文档。
- 已删除的文档。
- 未完成索引或索引失败的文档。

#### 权限控制

- 所有用户相关操作都必须基于当前登录用户身份判断权限。
- 前端隐藏按钮不能作为权限控制依据，后端必须独立校验权限。
- 普通用户访问其他用户资源时应返回无权限或不存在，不能泄露敏感信息。
- 问答检索必须在后端限制文档范围，不能依赖前端传入的文档 ID 作为唯一依据。
- 用户删除、文档删除、可见性变更后，后续问答必须立即遵守新的权限范围。

### 系统自动清理规则

#### 删除用户

删除用户时，系统必须删除该用户相关的所有内容：

- 用户资料。
- 用户聊天记录。
- 用户聊天消息。
- 用户上传的所有文档。
- 用户文档的索引内容。
- 用户上传文件的存储对象。
- 用户对其他公开文档的选择关系。
- 其他用户对该用户公开文档的选择关系。

#### 删除文档

删除文档时，系统必须删除该文档相关的所有内容：

- 文档记录。
- 文档索引内容。
- 文档上传文件。
- 所有用户对该文档的选择关系。

#### 历史引用

- 历史聊天消息可以保留已有回答文本和引用快照。
- 如果引用的文档已被删除，界面或接口需要能表达该来源已不可用。

### 验收标准

- `user1@test.com` 创建的聊天，`user2@test.com` 看不到也无法通过 URL/API 访问。
- `user1@test.com` 上传的私有文档，`user2@test.com` 看不到，也不会被问答检索使用。
- `user1@test.com` 上传的公开文档，`user2@test.com` 可以在公开文档列表中看到。
- `user2@test.com` 未勾选 `user1@test.com` 的公开文档时，问答不会使用该文档。
- `user2@test.com` 勾选 `user1@test.com` 的公开文档后，问答可以使用该文档。
- `user2@test.com` 取消勾选后，后续问答不再使用该文档。
- 普通用户无法删除其他用户的文档。
- 管理员可以删除任意用户的文档。
- 管理员不能删除自己的账号。
- 管理员不能查看其他用户的聊天内容。
- 管理员删除某个公开文档后，所有用户都不能再检索到该文档。
- 管理员删除某个用户后，该用户的聊天、文档、索引和公开文档选择关系都被清理。
- 不同用户可以上传内容相同的文件。
- 普通用户不能修改自己的邮箱或密码。
- 删除用户和删除文档不要求生成审计日志。
- 登录页展示默认测试账号和密码。
