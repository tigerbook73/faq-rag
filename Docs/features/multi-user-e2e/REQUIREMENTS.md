# multi-user-e2e 需求

## 1. 目标

为 multi-user feature 补充完整的三账号端到端自动化测试，覆盖 `multi-user/DESIGN.md` 2.15.2 节列出的核心 E2E 场景。

现有 Playwright 测试（`e2e/basic.test.ts`）只覆盖了登录页默认账号展示，不覆盖任何权限隔离场景。本 feature 补齐这个缺口。

---

## 2. 测试账号

使用现有 seed 账号：

- `admin@test.com` / `admin@123`（管理员）
- `user1@test.com` / `user1@123`（普通用户）
- `user2@test.com` / `user2@123`（普通用户）

---

## 3. 需要覆盖的场景

### 3.1 会话隔离

- `user1` 创建聊天，`user2` 登录后看不到该聊天。
- `user2` 直接访问 `user1` 聊天的 URL，无法加载内容（404 或空）。

### 3.2 私有文档隔离

- `user1` 上传私有文档，`user2` 的文档列表中看不到该文档。
- `user2` 直接调用 `DELETE /api/documents/{id}` 删除 `user1` 的文档，返回 404 或 403。

### 3.3 公开文档选择流程

- `user1` 上传并设置公开文档后，`user2` 在 Knowledge 页面的公开文档列表中可以看到该文档。
- `user2` 勾选前，该文档的内容不能出现在 `user2` 的问答回答中。
- `user2` 勾选后，该文档的内容可以出现在 `user2` 的问答回答中。
- `user2` 取消勾选后，该文档的内容不再出现在 `user2` 的问答回答中。

### 3.4 管理员操作

- 管理员可以在 Admin 页面看到所有用户和全站文档。
- 管理员删除某个用户后，该用户无法再登录（或登录后无法获得业务资料）。
- 管理员不能在 Admin 页面删除自己（确认按钮不可用或删除返回错误）。
- 管理员删除某个公开文档后，其他用户的公开文档列表中不再出现该文档。

---

## 4. 验收标准

- 上述所有场景均有对应的 Playwright 测试用例，且在本地可稳定运行。
- 测试不依赖 LLM 真实调用——涉及问答的场景（3.3）使用预置文档和关键词断言，或 mock LLM 响应。
- 测试可在每次 PR 合并前手动执行，或在具备 Supabase 测试环境时自动执行。
- 现有 `e2e/basic.test.ts` 的 2 个测试保持通过，不引入回归。

---

## 5. 不在范围内

- 性能或负载测试。
- 跨浏览器兼容性（只要求 Chromium）。
- 移动端适配验证（移动端测试另行计划）。
- 完整 LLM 检索质量评估（只验证权限边界，不验证回答质量）。
